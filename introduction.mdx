designlang is a CLI tool that reverse-engineers the design system of any live website. Point it at a URL and it produces 8 ready-to-use output files: AI-optimized markdown, W3C design tokens, Tailwind config, CSS custom properties, Figma variables, React theme, shadcn/ui theme, and a visual HTML preview.

## Get started in 30 seconds

<Steps>
  <Step title="Run the extractor">
    No install required — npx pulls the latest version automatically.

    ```bash
    npx designlang https://stripe.com
    ```
  </Step>
  <Step title="Get everything at once">
    Add `--full` to also capture screenshots, responsive breakpoints, and interaction states.

    ```bash
    npx designlang https://stripe.com --full
    ```
  </Step>
  <Step title="Use the output">
    Drop the generated files directly into your project — Tailwind config, CSS variables, Figma variables, and more are all ready to use.
  </Step>
</Steps>

## Output files

Every extraction produces 8 files:

| File | Description |
|------|-------------|
| `*-design-language.md` | AI-optimized markdown — feed to any LLM to recreate the design |
| `*-preview.html` | Visual report with color swatches, type scale, shadow gallery, and a11y score |
| `*-design-tokens.json` | [W3C Design Tokens](https://design-tokens.github.io/community-group/format/) format |
| `*-tailwind.config.js` | Drop-in Tailwind CSS theme extension |
| `*-variables.css` | CSS custom properties |
| `*-figma-variables.json` | Figma Variables import with dark mode support |
| `*-theme.js` | React/CSS-in-JS theme for Chakra, Stitches, Vanilla Extract |
| `*-shadcn-theme.css` | shadcn/ui `globals.css` variables |
