---
title: "Get started extracting design systems with designlang"
description: "Go from URL to 8 design output files in under a minute — designlang extracts colors, typography, shadows, and more, no setup required."
---

designlang reverse-engineers the design language of any live website and writes 8 ready-to-use files to disk — including an AI-optimized markdown document, W3C design tokens, a Tailwind config, CSS custom properties, Figma variables, a React theme, a shadcn/ui theme, and a visual HTML preview. You do not need a config file, an API key, or an account. Just a URL.

<Note>
  designlang uses Playwright to launch a headless Chromium browser. When you run it for the first time via `npx`, Playwright installs Chromium automatically. This download is ~150 MB and only happens once.
</Note>

## The fastest path

<Steps>
  <Step title="Run the extractor">
    Pass any URL to `designlang`. No install required — `npx` fetches the latest version automatically.

    <CodeGroup>

    ```bash npx (no install)
    npx designlang https://stripe.com
    ```

    ```bash Global install
    npm install -g designlang
    designlang https://stripe.com
    ```

    </CodeGroup>
  </Step>

  <Step title="Get everything at once">
    Add `--full` to also capture component screenshots, responsive behavior across 4 viewports, and hover/focus/active interaction states.

    ```bash
    npx designlang https://stripe.com --full
    ```
  </Step>

  <Step title="Use the output files">
    All files are written to `./design-extract-output/` by default. Each file is named after the URL — for example, extracting `vercel.com` produces:

    ```
    design-extract-output/
    ├── vercel-com-design-language.md
    ├── vercel-com-design-tokens.json
    ├── vercel-com-tailwind.config.js
    ├── vercel-com-variables.css
    ├── vercel-com-preview.html
    ├── vercel-com-figma-variables.json
    ├── vercel-com-theme.js
    └── vercel-com-shadcn-theme.css
    ```

    Drop any of these files directly into your project. The Tailwind config is a ready-to-use theme extension; the CSS file is a complete set of custom properties; the Figma JSON imports directly into Figma Variables.
  </Step>
</Steps>

## What you get (8 files)

| File | What it is |
|------|------------|
| `*-design-language.md` | AI-optimized markdown — feed it to any LLM to recreate the design |
| `*-preview.html` | Visual report with color swatches, type scale, shadow gallery, and a11y score |
| `*-design-tokens.json` | [W3C Design Tokens](https://design-tokens.github.io/community-group/format/) format |
| `*-tailwind.config.js` | Drop-in Tailwind CSS theme extension |
| `*-variables.css` | CSS custom properties |
| `*-figma-variables.json` | Figma Variables import with dark mode support |
| `*-theme.js` | React/CSS-in-JS theme for Chakra, Stitches, Vanilla Extract |
| `*-shadcn-theme.css` | shadcn/ui `globals.css` variables |
